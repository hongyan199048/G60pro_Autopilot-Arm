# 代价地图障碍物残留问题修复

**时间**：2026-04-29  
**问题**：固定障碍物的膨胀区不会消失，哪怕小车已经远离

---

## 问题根因

### Clearing 机制未启用

**配置问题**：
```yaml
# 修复前
single_1:
  clearing: false  # ❌ 只能标记障碍物，不能清除
  marking: true

single_2:
  clearing: false  # ❌ 只能标记障碍物，不能清除
  marking: true
```

**影响**：
- 单线雷达检测到的障碍物会一直保留在代价地图中
- 即使小车远离障碍物，膨胀区也不会消失
- 导致代价地图中出现"幽灵障碍物"

---

## Clearing 机制原理

### 射线追踪清除

```
激光雷达位置 ──────射线追踪────────→ 障碍物
     |                                    |
     └─── 清除路径上的旧障碍物 ──────────┘
          (0 到 raytrace_max_range)
```

**工作流程**：

1. **marking = true**：激光检测到障碍物 → 标记为障碍物（cost = 254）
2. **clearing = true**：激光射线路径上的空间 → 清除旧障碍物（cost = 0）

**示例场景**：

```
时刻 T1：小车在位置 A
  ┌─────┐
  │ 车  │ ──→ 激光 ──→ ■ 障碍物（标记）
  └─────┘

时刻 T2：小车移动到位置 B
                    ┌─────┐
  ■ 旧障碍物 ←── 激光 ──│ 车  │
                    └─────┘
  
  - clearing = true  → 旧障碍物被清除 ✓
  - clearing = false → 旧障碍物保留 ✗（问题所在）
```

---

## 为什么之前设置 clearing = false？

**原因**：单线雷达用于 **近场补盲**，避免误清除 Helios16 检测到的远距离障碍物。

**逻辑**：
- Helios16（16 线）：主力传感器，负责远距离检测 + 清除
- 单线雷达（1 线）：补盲传感器，只负责近场标记

**问题**：
- 单线雷达视野有限（1 线），但也会扫过远处空间
- 如果 clearing = false，单线雷达标记的障碍物永远不会被清除
- 导致代价地图中积累大量"幽灵障碍物"

---

## 修复方案

### 启用单线雷达的 clearing

```yaml
# 修复后
single_1:
  clearing: true   # ✓ 可以清除障碍物
  marking: true
  raytrace_max_range: 3.0  # 清除范围 3.0m
  obstacle_max_range: 2.5  # 标记范围 2.5m

single_2:
  clearing: true   # ✓ 可以清除障碍物
  marking: true
  raytrace_max_range: 3.0
  obstacle_max_range: 2.5
```

**关键参数**：
- `raytrace_max_range: 3.0`：射线追踪最大距离（清除范围）
- `obstacle_max_range: 2.5`：障碍物标记最大距离

**工作机制**：
1. 0-2.5m 范围内检测到障碍物 → 标记（marking）
2. 0-3.0m 范围内的射线路径 → 清除旧障碍物（clearing）
3. 2.5-3.0m 之间：只清除，不标记（避免误报）

## 注意事项

### 1. 为什么不担心误清除？

**问题**：单线雷达视野窄（1 线），会不会误清除 Helios16 检测到的障碍物？

**答案**：不会，原因如下：

1. **多传感器融合**：
   - Helios16（10Hz）+ 单线雷达（20Hz）同时更新代价地图
   - 即使单线雷达清除了某个栅格，Helios16 会立即重新标记

2. **更新频率**：
   - 局部代价地图更新频率：10Hz
   - 只要有任何一个传感器检测到障碍物，就会被标记

3. **射线追踪范围**：
   - 单线雷达只清除 0-3.0m 范围内的射线路径
   - 不会影响 3.0m 以外的障碍物

### 2. 静态地图障碍物会被清除吗？

**答案**：不会。

- **全局代价地图** = static_layer（静态地图）+ obstacle_layer（动态障碍物）
- static_layer 的障碍物不会被 clearing 清除
- 只有 obstacle_layer 的动态障碍物会被清除
