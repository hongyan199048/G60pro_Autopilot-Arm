# 避障功能修复说明

**修复时间**：2026-04-29  
**修复人员**：Claude (Tim_G60Pro)

## 修复的问题

### 1. ✅ 启用 Helios16 多线雷达（严重问题）

**问题**：局部和全局代价地图只使用单线雷达，主力传感器 Helios16 被禁用。

**修复**：
- 文件：`src/robot_navigation/config/nav2_params_real.yaml`
- 行号：121, 194
- 修改前：`observation_sources: single_1 single_2`
- 修改后：`observation_sources: scan single_1 single_2`

**影响**：恢复 Helios16 多线雷达（16 线，30m 量程）到避障系统，大幅提升远距离障碍物检测能力。

---

### 2. ✅ 调整单线雷达最小距离

**问题**：单线雷达 `obstacle_min_range: 0.6m`，无法检测 0.6m 以内的障碍物（如充电桩）。

**修复**：
- 文件：`src/robot_navigation/config/nav2_params_real.yaml`
- 行号：146, 162, 219, 235（4 处，局部/全局代价地图各 2 处）
- 修改前：`obstacle_min_range: 0.6  # B 测试：过滤近场自回波`
- 修改后：`obstacle_min_range: 0.3  # 降低到 0.3m，检测充电桩等近距离障碍物`

**影响**：充电对接时可检测到 0.3-0.6m 范围内的充电桩本体。

---

### 3. ✅ 删除全局代价地图重复配置

**问题**：`single_2` 配置重复了 6 行（238-244 行）。

**修复**：
- 文件：`src/robot_navigation/config/nav2_params_real.yaml`
- 行号：239-244
- 删除重复的：
  ```yaml
  raytrace_max_range: 3.0
  raytrace_min_range: 0.0
  obstacle_max_range: 2.5
  obstacle_min_range: 0.3
  observation_persistence: 0.5
  expected_update_rate: 20.0
  ```

**影响**：清理冗余配置，避免混淆。

---

### 4. ✅ 优化 pointcloud_to_laserscan 高度范围

**问题**：高度范围过窄（-1.4m 到 0.5m），可能过滤掉高/低障碍物。

**修复**：
- 文件：`src/robot_navigation/launch/navigation_real.launch.py`
- 行号：57-58
- 修改前：
  ```python
  'min_height': -1.4,  # 过滤地面（地面在 -1.54m）
  'max_height': 0.5,   # 过滤天花板，保留障碍物高度
  ```
- 修改后：
  ```python
  'min_height': -1.5,  # 更接近地面（地面在 -1.54m）
  'max_height': 1.0,   # 提高上限到 1.0m，检测更高障碍物
  ```

**影响**：扩大障碍物检测高度范围，减少漏检。

---

### 5. ✅ 增加 BaseObstacle 避障权重

**问题**：`BaseObstacle.scale: 0.02` 权重过低，避障不够激进。

**修复**：
- 文件：`src/robot_navigation/config/nav2_params_real.yaml`
- 行号：92
- 修改前：`BaseObstacle.scale: 0.02`
- 修改后：`BaseObstacle.scale: 0.05  # 提高避障权重，更激进规避障碍物`

**影响**：DWB 局部规划器更倾向于选择远离障碍物的轨迹。
